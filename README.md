DatabaseProxy (прокси для БД):
Ленивая инициализация — реальная БД создается только при первом запросе (экономия ресурсов)
Кеширование — повторные запросы возвращают данные из кеша, без обращения к БД
Контроль доступа — блокирует чтение/запись ключей с "admin"/"secret"
Инвалидация кеша — при записи данных кеш по ключу очищается

ImageProxy (прокси для изображений):
Ленивая загрузка — изображение загружается с диска только при первом вызове Display()
Информация без загрузки — GetInfo() возвращает данные о файле без реальной загрузки

В Main():
Создаются прокси, реальные объекты не создаются
Первый вызов → инициализация реального объекта + выполнение операции
Повторный вызов → данные из кеша (БД) или уже загруженный объект (изображение)
Демонстрируется запрет доступа к секретным данным
Прокси перехватывает вызовы и добавляет логику (кеширование, контроль доступа, ленивую загрузку) без изменения реального объекта
